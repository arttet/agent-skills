# Contributing to Agent Skills

Thank you for your interest in contributing! This document covers how to submit changes to the repository.

## How to Contribute

1. **Fork the Repository** — create your own copy of `arttet/agent-skills`.
2. **Create a Branch** — use a descriptive branch name (e.g., `feat/my-new-skill` or `fix/skill-name`).
3. **Make Your Changes** — follow the conventions described in the documentation.
4. **Test Your Changes** — run `just validate` and `just evaluate` to check quality.
5. **Submit a Pull Request** — open a PR with a clear description of what changed and why.

## Describing Your Changes

When opening a PR, explain:

- **What problem does your change solve?**
- **What is the scope?** (new skill, bug fix, documentation improvement)
- **How did you test it?**

## Questions?

If you are unsure about anything, read the [documentation](https://skills.arttet.dev/) first. It covers skill format, validation, and the development workflow.
